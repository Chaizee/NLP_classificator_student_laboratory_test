# Sentiment Analysis of Russian Geo-Reviews (Cafes & Restaurants)

Проект по **классификации тональности отзывов** о заведениях общественного питания на русском языке с использованием методов NLP и машинного обучения.

---

## Описание задачи

### Цель
Предсказать эмоциональную окраску текстового отзыва о кафе или ресторане:

| Класс | Рейтинг | Описание |
|-------|---------|----------|
| **Negative** | 1-2 | Негативный отзыв |
| **Neutral** | 3 | Нейтральный отзыв |
| **Positive** | 4-5 | Позитивный отзыв |

### Задачи
- Провести разведочный анализ данных (EDA)
- Очистить и предобработать тексты на русском языке
- Реализовать различные методы векторизации текста
- Обучить и сравнить классические ML-модели и нейросети
- Выбрать лучшую модель по метрике F1-macro

---

## Источник данных

**Датасет:** [d0rj/geo-reviews-dataset-2023](https://huggingface.co/datasets/d0rj/geo-reviews-dataset-2023)

- **Формат:** Apache Parquet
- **Язык:** Русский
- **Объём:** ~92,000 отзывов
- **Категории:** Кафе и рестораны

---

## Что было сделано

### 1. Разведочный анализ данных (EDA)

- **Загрузка данных** из Hugging Face Hub
- **Очистка данных:**
  - Удалён 1 дубликат
  - Исключены записи с рейтингом `0` (ошибки разметки)
  - Удалены отзывы > 5000 символов (спам/реклама)
- **Анализ распределения:**
  - Выявлен сильный дисбаланс классов
  - Негативные отзывы в среднем длиннее позитивных
  - Корреляция Спирмена: отрицательная (чем ниже рейтинг, тем длиннее текст)

### 2. Предобработка текста

- **Очистка текста**
  - Удаление URL, @упоминаний, #хештегов
  - Удаление спецсимволов и цифр
  - Приведение к нижнему регистру

- **Лингвистическая обработка**
  - Токенизация
  - Лемматизация (pymorphy3)
  - Удаление стоп-слов (nltk)
  - Фильтрация токенов < 3 символов

- **Фильтрация**
  - Удаление пустых и слишком коротких отзывов

### 3. Векторизация текста

| Метод | Описание | Размерность | Инструмент |
|-------|----------|-------------|------------|
| **TF-IDF** | Term Frequency-Inverse Document Frequency, n-gram (1,2) | 5000 | `sklearn.TfidfVectorizer` |
| **Word2Vec** | Skip-gram, усреднение векторов слов | 100 | `gensim.Word2Vec` |
| **FastText** | С учётом подслов (subwords) | 100 | `gensim.FastText` |
| **RuBERT** | Multilingual BERT, mean pooling | 768 | `DeepPavlov/rubert-base-cased` |

> **RuBERT** применялся на подвыборке из 5000 отзывов из-за вычислительной сложности.

### 4. Работа с дисбалансом классов

- **Стратифицированное разделение:** train/test (80/20)
- **Балансировка:** Undersampling до минимального класса
- **Label Encoding:** negative=0, neutral=1, positive=2

### 5. Обучение моделей

#### Классические алгоритмы ML:
1. **Logistic Regression** (TF-IDF)
   - `class_weight='balanced'`, max_iter=1000
   
2. **Multinomial Naive Bayes** (TF-IDF)
   - Быстрый базовый классификатор
   
3. **Random Forest** (Word2Vec)
   - 100 деревьев, `class_weight='balanced'`
   
4. **CatBoost** (TF-IDF)
   - 500 итераций, depth=6, loss_function='MultiClass'

#### Нейросетевые архитектуры (PyTorch):

**5. MLP (Multi-Layer Perceptron)**
    `Input (100)` → `Dense (128)+ReLU` → `Dense (64)+ReLU` → `Dense (3) + Softmax`
    
**6. SimpleRNN (LSTM)**
    `Input (100)` → `LSTM (hidden=64)` → `Dense (3) + Softmax`


### 6. Оценка качества

**Метрики:**
- Accuracy (точность)
- Precision (macro) — точность по каждому классу
- Recall (macro) — полнота по каждому классу
- **F1-Score (macro)** — основная метрика выбора

**Дополнительно:**
- Classification report для лучшей модели
- Confusion Matrix (матрица ошибок)
- Визуализация сравнения моделей

## Результаты

### Сравнение моделей

| Модель | Векторизация | Accuracy | Precision | Recall | **F1-Macro** |
|--------|--------------|----------|-----------|--------|--------------|
| CatBoost | TF-IDF | 0.82 | 0.60 | 0.72 | 0.63 |
| Logistic Regression | TF-IDF | 0.84 | 0.63 | 0.73 | 0.65 |
| Random Forest | Word2Vec | 0.82 | 0.63 | 0.71 | 0.63 |
| MLP | Word2Vec | 0.83 | 0.65 | 0.73 | 0.65 |
| Naive Bayes | TF-IDF | 0.82 | 0.60 | 0.72 | 0.63 |
| RNN (LSTM) | Word2Vec | 0.83 | 0.65 | 0.73 | 0.65 |

> **Лучшая модель:** `CatBoost + TF-IDF`  
> **Основная метрика:** F1-Score (macro)

### Ключевые выводы

1. **CatBoost** показал наилучший результат благодаря работе с разреженными признаками TF-IDF
2. **Трансформеры (RuBERT)** требуют значительных вычислительных ресурсов, но могут дать прирост качества на больших данных
3. **Дисбаланс классов** критически влияет на качество — необходима балансировка
4. **Негативные отзывы** содержат больше деталей и длиннее позитивных
